# Think In Agents

- Each agent should be responsible for one thing (high cohesion, low coupling).
- Break up a problem into simple agents, for example:
    - Classifier agent
    - Processor agent
    - Response agent

# Use Mock Data

- Mock APIs
- Hardcode small parts
- Focus on flow between components

# Use Copilot Iteratively 

- Don't try to build the whole system in one go, iteratively plan and design before beginning to code. 
- Build -> Refine -> Integrate

# Focus On Demo

- The app should work within a 1 minute demo
- Focus on the main application flow

# Winning Formula

- Clear architecture
- Agent-based design
- Azure integration
- Working demo
- Clean explanation
